## Open Source OSINT Tools

All tools are written in **Go 1.22+**, designed for speed, explicit control, and pipeline-friendly output. Most require no API key and no configuration files.

  <Card title="Zero Config" icon="zap" href="/open-source/ip-thc">

  </Card>

    Structured, line-oriented output designed for grep, awk, and automation.
  </Card>
  <Card title="Proxy & TLS" icon="shield" href="/open-source/brave-search">

### Tool Comparison

| Tool | API Key Required | Data Source | Primary Use Case |
|------|:---:|------------|-----------------|
| [ip-thc](/open-source/ip-thc) | No | ip.thc.org | Reverse DNS, subdomains, CNAME |
| [search-leaks](/open-source/search-leaks) | No | Hudson Rock | Breach/leak statistics |
| [reverse-whois](/open-source/reverse-whois) | Yes (WhoisXML) | WhoisXML API | Reverse WHOIS enumeration |
| [sub-alter](/open-source/sub-alter) | Yes (WhoisXML) | WhoisXML API | Domain/subdomain discovery |
| [brave-search](/open-source/brave-search) | Yes (Brave) | Brave Search | URL extraction from search |

## Mail Forwarding Service

A complete, free, public mail forwarding service at [forward.haltman.io](https://forward.haltman.io).

### Key characteristics

- **Stateless** — no mailbox storage, forwarding only
- **Abuse-aware** — email confirmation, rate limiting, anti-spoofing
- **Fully open-source** — no open-core, no artificial limitations, no telemetry
- **Self-hostable** — complete documentation to deploy on your own infrastructure

### Stack Components

| Component | Technology | Role |
|-----------|-----------|------|
| [Core Stack](/mail-forwarding-selfhost/mail-forwarding-core) | Postfix, MariaDB, PostSRSd, OpenDKIM | SMTP forwarding engine |
| [API](/mail-forwarding-selfhost/mail-forwarding-api) | Node.js, Express | Alias management endpoints |
| [UI](/mail-forwarding-selfhost/mail-forwarding-ui) | Next.js, React, TailwindCSS | Web frontend for users |
| [Cleanup](/mail-forwarding-selfhost/cleanup-cronjob) | Bash, cron | Expired token purge |

## Design Principles

<ExpandableGroup>
  <Expandable title="No magic, no abstractions" default-open="true">
    All projects are intentionally simple and transparent. If you understand the underlying technology (SMTP, DNS, HTTP), you can understand and trust the implementation.
  </Expandable>

  <Expandable title="No vendor lock-in" default-open="false">
    Everything is open-source. No proprietary dependencies. Standard protocols and widely available tools.
  </Expandable>

  <Expandable title="Explicit over implicit" default-open="false">
    All behavior is controlled via CLI flags or environment variables. No hidden defaults, no environment sniffing, no telemetry.
  </Expandable>
</ExpandableGroup>
