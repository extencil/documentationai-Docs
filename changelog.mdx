## Changelog Overview

Stay up to date with the latest releases of Haltman. This changelog highlights new features, improvements, bug fixes, and any breaking changes. You receive notifications for updates through your dashboard at `https://dashboard.example.com`.

<Update label="2024-10-15" description="v1.2.0" tags={["feature", "improvement"]}>

## New Features

- Added real-time collaboration for documentation pages, enabling multiple users to edit simultaneously.
- Introduced version history with diff viewer to track changes over time.

## Improvements

- Enhanced search functionality with fuzzy matching and full-text indexing.
- Optimized page load times by `>40%` through lazy loading components.

## Bug Fixes

- Fixed rendering issues with nested lists in markdown editor.
- Resolved authentication token refresh problems during long sessions.

</Update>

## Upgrade Process

Follow these steps to upgrade Haltman to the latest version.

<Steps>
  <Step title="Backup Your Data" icon="database">

  Create a full backup of your workspace.

  ```
  haltman backup --output=/path/to/backup
  ```

  </Step>
  <Step title="Update Package" icon="download">

  ```
  npm install haltman@latest
  ```

  Or use your preferred package manager.

  </Step>
  <Step title="Run Migrations" icon="settings">

  ```
  haltman migrate
  ```

  Review any migration notes in the release.

  </Step>
  <Step title="Restart Service" icon="refresh-cw">

  Restart your Haltman instance and verify functionality.

  </Step>
</Steps>

## Migration Guides

<ExpandableGroup>
  <Expandable title="v1.2.0 Migration" default-open="true">

  No breaking changes in this release. Simply run the upgrade steps above.

  If you encounter issues with real-time collaboration, ensure your firewall allows WebSocket connections on port 443.

  </Expandable>
  <Expandable title="v1.1.0 Breaking Changes">

  Webhook endpoints changed from `/v1/webhooks` to `/v2/webhooks`. Update your integrations accordingly.

  ```javascript
  // Old
  fetch('https://api.example.com/v1/webhooks', { ... });

  // New
  fetch('https://api.example.com/v2/webhooks', { ... });
  ```

  </Expandable>
</ExpandableGroup>
